No contexto do Git, uma "tag" é uma referência específica a um ponto na história do repositório. Elas são usadas para marcar commits importantes, como versões de software estáveis. Existem dois tipos principais de tags no Git:

  1. Tags Leves (Lightweight Tags): São simples apontadores para um commit específico. Elas são criadas usando o comando git tag nome_da_tag. Essas tags são úteis para marcar pontos importantes na história do repositório de forma leve, sem informações adicionais.
  2. Tags Anotadas (Annotated Tags): São mais elaboradas e incluem informações como nome do autor, endereço de e-mail, data e uma mensagem descritiva. Para criar tags anotadas, você usa o comando git tag -a nome_da_tag -m "Mensagem descritiva".

As tags são úteis para identificar rapidamente pontos significativos no histórico do Git e são frequentemente usadas para marcar lançamentos de software. Elas não se movem à medida que novos commits são feitos, tornando-as estáveis como referências a versões específicas
