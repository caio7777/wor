Entendendo a diferença dos estágios do git

Para começar é bom entender quando um arquivo está: untracked/tracked/modified.

Untracked
É o estado que se encontra o arquivo, que está no work diretory mas não está sendo trakeado ou seja rastreado pelo git.


tracked
Este estado é onde ele se encontra rastreado para nosso repositorio local, seu sub-objetivo e ser commitado(a) para que assim possa cumprir com o objetivo final que e onde todas as mudanças foram commitadas e por fim foram enviadas/sincronizadas com as informaçoes do repositorio remoto!


modified
por exemplo: untraked => /antes do estaged... tracked => /quando em staged... modfied => Então uma observação que eu pude fazer é que, um arquivo modified, é tanto tracked quanto modfied pois para estar modified ele precisa está tracked...

about stash

O comando stah é uma forma de limpar o diretorio de trabalho e gravar uma copia das ultimas alterações feitas após o ultimo commit, lhe dando a possibilidade de tomar outra abordagem ao objetivo que se deseja alcançar, salvando a abordagem anterior e caso queira voce pode voltar continuar daquela abordagem que voce havia deixado no stash para uma futura possibilidade!

mais sobre stash

e normal as stash poder ser acessada em um branch que nao seja onde ela foi criada?

Sim, é normal que uma stash possa ser acessada em um branch diferente daquele em que foi criada. A stash é uma funcionalidade do Git que permite que você armazene temporariamente as alterações que fez em sua cópia de trabalho para poder alternar para outro branch ou tarefa sem perder seu trabalho não commitado. Quando você cria uma stash, ela fica associada ao repositório Git, não a um branch específico. Isso significa que você pode aplicar uma stash em qualquer branch do mesmo repositório. Para fazer isso, você pode usar comandos como git stash apply ou git stash pop seguido do nome da stash que deseja aplicar. Se você não especificar um branch, a stash será aplicada no branch atual. Portanto, a capacidade de acessar uma stash em um branch diferente é uma característica útil do Git, permitindo que os desenvolvedores gerenciem melhor suas alterações em diferentes partes de um projeto.
