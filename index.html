Entendendo a diferença dos estágios do git

Para começar é bom entender quando um arquivo está: untracked/tracked/modified.

Untracked
É o estado que se encontra o arquivo, que está no work diretory mas não está sendo trakeado ou seja rastreado pelo git.


tracked
Este estado é onde ele se encontra rastreado para nosso repositorio local, seu sub-objetivo e ser commitado(a) para que assim possa cumprir com o objetivo final que e onde todas as mudanças foram commitadas e e por fim foram enviadas/sincronizadas com as informaçoes do repositorio remoto!


modified
por exemplo: untraked => /antes do estaged... tracked => /quando em staged... modfied => Então uma observação que eu pude fazer é que, um arquivo modified, é tanto tracked quanto modfied pois para estar modified ele precisa está tracked...
